# 2026-Algorithm-Study 운영 규칙

### 1. 기본 원칙
* **언어 및 난이도:** 제한 없음 (본인의 수준에 맞춰 자유롭게 선택)
* **파일명 규칙:** 자유
* **파일 경로:** `본인 폴더/주차/코드파일`
  * 예시: `yujin/week01/14599.cpp`
* **벌금 및 페널티:** 현재 논의 중 (확정 시 업데이트 예정)

---

### 2. 커밋 컨벤션 (Commit Convention)

커밋 메시지는 아래 규칙에 따라 작성하며, `git commit -m` 명령어를 사용합니다.

* **형식:** `태그: [플랫폼]_문제번호_문제이름`
* **예시:** `feat: [BOJ]_1000_A+B`
* **추가 설명(선택):** `git commit -m "feat: [BOJ]_1000_A+B" -m "간략한 설명"`

#### 💡 커밋 태그
| 태그 | 설명 |
| :--- | :--- |
| **feat** | 새로운 문제 풀이 추가 |
| **fix** | 틀린 풀이 수정 |
| **refactor** | 코드 리팩토링 |
| **comment** | 주석 추가(코드 변경 X) 혹은 오타 수정 |
| **docs** | README와 같은 문서 수정 |
| **merge** | 머지 시 사용 |
| **rename** | 파일, 폴더명 수정 혹은 이동 |

#### 💡 플랫폼 태그
| 플랫폼 | 태그 |
| :--- | :--- |
| **백준** | `[BOJ]` |
| **프로그래머스** | `[PGS]` |
| **리트코드** | `[LTC]` |
| **코드포스** | `[CFS]` |
| **삼성 SW Expert Academy** | `[SEA]` |
| **기타** | `[ETC]` |

---

### 3. PR (Pull Request) 규칙

* **PR 제목:** `날짜 이름 문제이름`
  * 예시: `260215 양유진 A+B`
* **PR 메시지:** 양식은 자유이나 **사용된 알고리즘**, **풀이 방법** 등을 작성하는 것을 권장합니다.
* **머지(Merge) 규칙:** * PR은 생성 **다음 날** 머지합니다. 
  * 다른 멤버들은 자유롭게 제안 사항이나 궁금한 점을 코멘트로 남길 수 있습니다.
